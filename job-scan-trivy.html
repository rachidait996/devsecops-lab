Job Trivy – Scan de vulnérabilités

Exploration approfondie des failles de sécurité dans les images Docker

1. Trivy : un scanner open source complet

Trivy est un outil de sécurité open source qui analyse les images Docker, les dépendances d'applications, les fichiers IaC (Infrastructure as Code), et plus encore. Il référence des bases de données comme Debian Security Tracker, Red Hat CVE, ou NVD pour identifier les CVE présentes.

2. Types de vulnérabilités détectées

  • Packages systèmes : vulnérabilités dans les bibliothèques C (ex: glibc, openssl)
  • Dépendances applicatives : failles dans les fichiers package.json, requirements.txt, etc.
  • Secrets exposés : fichiers contenant des jetons ou mots de passe codés en dur
  • Fichiers IaC (Terraform, Dockerfile, etc.) : erreurs de configuration pouvant mener à une compromission

3. Classement des CVEs

Trivy classe les CVE selon leur niveau de gravité (Low, Medium, High, Critical) en se basant sur le score CVSS. Cela permet aux équipes de prioriser les corrections.

Vulnérabilités système (OS Packages)

Ces failles affectent les bibliothèques système comme libssl ou zlib. Elles représentent une surface d'attaque critique car elles sont à la base de tout l'environnement.

  • CVE-2022-4450 sur libssl1.1 – vulnérabilité de chiffrement.
  • CVE-2023-0286 sur libcrypto1.1 – type confusion, fuite mémoire.

Failles dans les dépendances applicatives

Ces failles affectent les bibliothèques déclarées dans package.json, requirements.txt, etc. Souvent sous-estimées, elles peuvent permettre une exécution de code ou une élévation de privilège.

  • CVE-2020-7788 sur ini – Pollution de prototype.
  • CVE-2020-7610 sur bson – désérialisation non sécurisée.
  • CVE-2021-23358 sur underscore – injection de code.
